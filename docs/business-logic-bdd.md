# StickerFit Studio BDD Documents

This folder keeps the product behavior specification in Behavior-Driven Development format.

BDD writing conventions used here:

- Each behavior is written as `Feature -> Rule -> Scenario`.
- Each scenario uses `Given / When / Then / And` steps.
- Scenario IDs are stable and shared by the English and Chinese versions.
- Business terms are defined before scenarios so humans, AI agents, and future automated tests can resolve the same vocabulary.
- Future optimization items are also written as acceptance scenarios instead of vague roadmap notes.
